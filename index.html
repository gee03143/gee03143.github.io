ABOUT

소개

상용 게임 수준의 시스템 아키텍처를 직접 설계하고 구현하며 체득하는 것을 목표로 합니다. Inventory, Equipment, Weapon, Damage, Niagara FX, 자동화 테스트까지 — TPS 한 회차에서 마주치는 시스템의 골격을 스스로 쌓아올리는 방식으로 학습합니다.

Epic Games의 공식 샘플인 Lyra Starter Game의 설계 패턴을 참고하되, 단순 복제가 아닌 패턴의 목적을 이해한 뒤 도메인에 맞게 재구성하는 방향을 택하고 있습니다.

  • 01

    확장성 · 데이터 드리븐

    Fragment / Definition · Instance 분리로 콘텐츠 추가 비용 최소화

  • 02

    멀티플레이 권한 모델

    Server-authoritative · FastArraySerializer delta 복제

  • 03

    테스트 가능성 · 유지보수

    Component 분리 + Delegate loose coupling · 자동화 테스트

  • 04

    상용 아키텍처의 재해석

    Lyra 패턴을 도메인에 맞게 축소·재구성

UE 5.7 · C++ 멀티플레이어 TPS 시스템 학습 프로젝트

Inventory · Equipment · Weapon · Damage · Niagara FX · 자동화 테스트를 단일 코드베이스에 쌓아가며 상용 수준 아키텍처를 체득하는 개인 학습 프로젝트입니다. 서버 권위 모델과 FFastArraySerializer 기반 delta 복제, Fragment 패턴을 통한 데이터 드리븐 확장이 핵심 설계입니다.

  • 6개 시스템 · Inventory / Equipment / Weapon / Health / FX / Tests
  • 10건 자동화 테스트 · Inventory 5 + Equipment 5 (복제 시뮬 포함)
  • Lyra 패턴 재해석 · GAS 전 단계의 권한 모델을 직접 구축
